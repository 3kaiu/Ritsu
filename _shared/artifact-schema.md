# 产物契约 (Artifact Schema)
> Ritsu Bundle 共享协议 v2.1 · 定义所有跨技能中间产物的强制格式。
> 下游技能读取上游产物时，必须按此 Schema 解析，禁止自由解读。

---

## Schema 0：AGENTS.md（init → 全技能）

**文件路径**：`{项目根}/AGENTS.md`

```markdown
# {项目名} 核心开发约束 (Ritsu Layer 3)
ritsu-version: 2.1.0
domain: {frontend | backend | fullstack | infra | data}

## 1. 技术栈底座
- 语言/运行时: {真实值，禁止留空}
- UI 框架（如有）: {真实值 或 无}
- 状态管理/网络层: {真实值 或 无}

## 2. 架构与目录法则
- 分层模式: {MVC | MVVM | DDD | 其他}
- 核心模块依赖方向: {例：controller → service → repository}

## 3. 质量门禁 (Quality Gates)
- Lint: `{可直接执行的命令，如 npm run lint}`
- Test: `{可直接执行的命令，如 npm run test}`
> ⚠️ /r-dev 和 /r-review 完成后必须全量执行，跑绿才算交付。
> 若命令尚未建立，填写 `待补充` 并创建对应 Task。
```

## Schema 1：Handoff（think → dev / review）

**文件路径约定**：`{项目根}/ritsu/handoff-{kebab-slug}.md`

> `{kebab-slug}` 命名规则：取需求描述的前 3 个有效英文关键词，转 kebab-case 小写。
> 例：需求"用户登录流程" → `handoff-user-login-flow.md`
> 同名冲突时追加序号：`handoff-user-login-flow-2.md`

```markdown
# Handoff: {kebab-slug}
_generated by /r-think · domain: {value} · date: {YYYY-MM-DD}_

## 1. 边界与依赖
- **目标范围**（In Scope）: ...
- **Out of Scope**: ...（明确列出，禁止模糊）
- **新增依赖**: {名称 / 版本 / License / 体积} 或 `无`

## 2. 核心契约 (Contract)

### API / 接口契约（backend / fullstack 必填）
- Method + Path:
- Request Schema:
- Response Schema:
- 错误码清单:

### 数据模型（backend / fullstack 必填）
- 表名/集合名:
- 关键字段及类型:
- 索引设计:

### 组件契约（frontend / fullstack 必填）
- 组件名:
- Props 类型定义:
- Emits / 回调:
- 状态机（若有）:

## 3. 攻击测试防线
- **宕机响应**: 外部依赖 502 时的降级方案（不允许填"暂不处理"）
- **10x 瓶颈**: 流量放大 10 倍最先撑爆的点及缓解措施
- **回滚步骤**: 上线发现致命 Bug 的确切回滚指令（逐步骤列出）

## 4. 实施清单
- [ ] `{文件路径}`: {主干逻辑描述，精确到函数/组件级别}
- [ ] ...
> 每条必须精确到文件，确保 /r-dev 执行时无需做任何技术决策。
```

> ⚠️ 红线：不允许出现 `TODO`、`待定`、`后续完善`、`暂不处理` 等占位符。

## Schema 3：Review Stamp（review → 归档）

**文件路径约定**：`{项目根}/ritsu/review-stamp-{YYYYMMDD-HHMMSS}.md`

> ⚠️ Review Stamp **必须写入文件**，同时在会话末尾内联输出。triage 和 merge 决策依赖文件中的结论。

```markdown
# Review Stamp: {YYYYMMDD-HHMMSS}
_generated by /r-review v2.1.0 · domain: {value}_

## 结论
**{PASS | FAIL}**

## Hard Stop 命中
{无 | P{N} — {描述}}

## 领域发现
{无 | - 问题描述（严重度：高/中/低）}

## 溯源对账
{已对账: ritsu/handoff-{slug}.md | 无 Handoff 存档（风险已知悉）}

## 质量门禁执行结果
- Lint: {✅ 通过 | ❌ 失败，输出：...}
- Test: {✅ 通过 | ❌ 失败，输出：...}
```
