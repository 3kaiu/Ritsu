# Learning Engine Rules

- `scanHumanCorrections` is the shared entry point for scanning ctx diffs
- Heuristic rules (5 built-in patterns) are synthesized first, then LLM rules (if `RITSU_LLM_ENABLED=1`)
- LLM rules are generated by `llm-synthesizer.ts` via OpenAI-compatible API
- Similarity: unified `similarity.ts` with Jaccard (CJK support) + Cosine fallback
- Vector store: `bun:sqlite` in native-bridge.ts with char-n-gram hash embeddings
- mining-sheet → promotePreference → .ritsu/preferences.yaml → reconcilePreferences → rules/ast-grep/
