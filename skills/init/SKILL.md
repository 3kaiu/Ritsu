# Init: 架构级指纹识别与基线初始化
11: 
12: **触发条件**：用户输入 `/r-init`。
13: 
14: ## 执行流水线
15: 
16: ### 1. 递归指纹扫描与技术栈感知 (Recursive Fingerprinting)
17: 
18: 识别项目的底层架构基因：
19: - **技术栈识别**：通过递归扫描关键指纹文件（如 `package.json`, `go.mod`, `pubspec.yaml`, `charts/`）确定项目所属的多态领域。
20: - **工具发现**：自动识别已安装的工具库（如 `ahooks`, `tanstack`, `riverpod`），用于后续的人格微调。
21: - **架构模式**：识别 Mono-repo 或 Polyglot 结构。
22: 
23: ### 2. 架构级 AGENTS.md 生成
24: 
25: 建立项目质量与协作基线：
26: - **技术栈声明**：在 `AGENTS.md` 中显式标注检测到的技术栈组合。
27: - **质量门禁注入**：根据技术栈自动填充对应的测试与 Lint 命令。
28: - **人格锁定**：在 `AGENTS.md` 中预设该项目默认的“资深架构师”人格。
29: 
30: ### 3. 工程纯净度与 AI 过程配置
31: 
32: 配置 AI 工作空间环境：
33: - 确保 `.gitignore` 包含 `.ritsu/` 等 AI 过程目录。
34: - 初始化语义引擎所需的 `.ritsu/` 目录结构。
35: 
36: ### 4. 交付摘要与引导
37: 
38: > 引用 `_shared/skill-common-steps.md` Step 4（skill=init）
39: 
40: **引导建议**：
41: - 初始化完成后，告知用户已采用的人格和识别出的技术栈。
42: - **示例**：“项目已完成指纹识别（检测到 React + Node.js 全栈架构）。我已就绪，您可以运行 `/r-think` 开始需求评审。”
