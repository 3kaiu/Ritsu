# Domain: Data / ML (数据与机器学习)

> 继承 `_base.md` 全部内容。以下仅声明数据领域增量。

## 领域增量诊断假设方向
- 数据倾斜 / 分布漂移 (Data Drift)
- 特征工程管线断裂 / Schema 不兼容
- 训练-推理不一致 (Training-Serving Skew)
- 资源竞争 (GPU 显存 / 磁盘 IO)

## 领域增量编码纪律
- **数据管线幂等性**：ETL 步骤必须可重入，重复执行不产生脏数据。
- **实验可复现**：随机种子、超参数、数据版本必须锁定并记录。

## 领域增量审查攻击向量
- **数据泄露**：训练集是否包含了测试集信息 (Label Leakage)？
- **偏见与公平性**：模型输出是否存在系统性偏见？
- **隐私合规**：数据集是否包含 PII？是否满足 GDPR/数据脱敏要求？

## 领域增量 PR 合并要求
- 必须提供 **数据管线端到端验证报告** 或 **模型指标对比表**
- 检查是否引入了新的数据依赖或变更了特征 Schema

## 纪律-攻击向量对偶映射 (领域增量)
| 编码纪律 (防守) | 审查攻击向量 (测试防守) |
|---|---|
| 数据管线幂等性 | 重复执行 ETL 是否产生脏数据？ |
| 实验可复现 | 随机种子/超参数是否锁定？能否复现同一结果？ |
