# Domain: Frontend (前端)

> 继承 `_base.md` 全部内容。以下仅声明前端领域增量。

## 领域增量诊断假设方向
- 缓存过期 / Service Worker 拦截
- 闭包引起的旧值 (Stale Closure)
- 状态同步时机错误 (Race Condition in State)
- 三方库版本冲突 / Peer Dependency 缺失
- Hydration 冲突 (SSR 场景)
- CORS 拦截 / 资源加载 404

## 领域增量编码纪律
- **Re-render 检查**：状态变更粒度是否最小化？严禁滥用全局状态。
- **竞态条件**：异步网络请求必须做取消 (AbortController) 或防抖处理。
- **内存泄漏**：绑定的全局事件或定时器必须在组件销毁时解绑。

## 领域增量审查攻击向量
- **XSS 与数据注入**：用户输入是否直接渲染在 DOM 中？
- **状态篡改**：前端状态管理是否容易被直接从 Console 或插件中篡改并绕过业务校验？
- **性能与渲染**：是否存在不必要的大面积重渲染？大列表是否做了虚拟化？
- **跨域信任**：后端 API 返回值是否被前端无条件信任并渲染？

## 领域增量 PR 合并要求
- 必须提供 **UI 变更截图或录屏**
- 检查是否增加了不必要的大体积三方包

## 纪律-攻击向量对偶映射 (领域增量)
| 编码纪律 (防守) | 审查攻击向量 (测试防守) |
|---|---|
| Re-render 检查 | 性能与渲染：大面积重渲染？ |
| 竞态条件 (AbortController) | 状态同步时机错误：请求取消是否生效？ |
| 内存泄漏 (组件销毁解绑) | 全局事件/定时器是否仍在运行？ |
